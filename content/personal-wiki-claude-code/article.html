図2: Claude Code → Wiki 自動書き込みパイプライン

Stop hook: セッションの学びを自動保存

最も実用的なのはStop hookだ。3Claude Code Hooks 公式ドキュメント Claude Codeがレスポンスを完了するたびに発火し、そのセッションで学んだことや解決した問題をwikiに書き出せる。

// .claude/settings.json
{
  "hooks": {
    "Stop": [{
      "hooks": [{
        "type": "agent",
        "prompt": "セッションの作業内容を要約し、wiki/に適切なMarkdownファイルとして書き出してください",
        "timeout": 120
      }]
    }]
  }
}

agentタイプを使うのがポイント。サブエージェントがファイルシステムに直接アクセスして、Markdownファイルの作成・更新ができる。

PostToolUse hook: 調査結果の自動保存

Web検索やファイル編集の後に発火するhook。例えば新しいライブラリを調べた結果を、リサーチノートとして自動保存できる。3Claude Code Hooks 公式ドキュメント

Scheduled Tasks: 定期的な整理

wikiは書くだけではカオスになる。4Claude Code Scheduled Tasks 定期タスクで以下を自動実行する:

  • 孤立ノートの検出: どこからもリンクされていないノートを発見
  • MOC（目次ページ）の再生成: トピックごとのインデックスを最新化
  • リンク切れの修復: ファイル名変更で壊れたリンクを修正

フロントマターで構造化

statusフィールドの成熟度モデル（🌱seed → 🌿fern → 🌳evergreen）は、ノートの品質管理に有効。AIが定期的にリンク数や更新頻度を見て昇格させる。
