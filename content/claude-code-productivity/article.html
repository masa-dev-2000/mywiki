🔧 Layer 1: 基盤設定 — CLAUDE.md・フック・MCP

CLAUDE.md: プロジェクトの「知能」

CLAUDE.mdはプロジェクト固有のルール・慣習・コンテキストをAIに伝えるファイルだ。/initコマンドで自動生成し、段階的に洗練していくのが推奨パターン。2Claude Code 公式ドキュメント

ただし重要な注意点がある。CLAUDE.mdは「助言的」であり、遵守率は約80%にとどまる。1Anthropic 公式ベストプラクティス 200行以内に収め、本当に重要なルールに絞ることが効果を最大化するコツだ。

CLAUDE.mdの遵守率80%に対し、Hooksは確定的に100%実行される。2Claude Code 公式ドキュメント 「絶対に守らせたいルール」はHooksで実装するのが正解だ。

  • PostToolUse: ツール実行後のアクション（自動フォーマット、テスト実行など）
  • Stop: 応答完了時のアクション（自動コミット、ログ記録など）

🗺️ 実践ロードマップ

全部を一度に導入する必要はない。段階的に積み上げていくアプローチが推奨される。1Anthropic 公式ベストプラクティス

  1. 今すぐ（15-30分）: /initでCLAUDE.mdを自動生成し、200行以内に洗練する。基本的なHook（自動フォーマット等）を設定する。
  2. 今週中: MCP接続を設定する（GitHub、データベース等）。カスタムスキルを2-3個作成して、ドメイン知識を分離する。
