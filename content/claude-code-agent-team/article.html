Agent Teams ― 協調するマルチエージェント

Agent Teamsは、独立したClaude Codeセッション同士が直接通信・協調することで、Subagentsでは不可能だった「議論・挑戦・合意形成」を実現する仕組みだ。5Claude Code Agent Teams 公式ガイド

アーキテクチャと設定

Agent TeamsはTeam Lead + Teammates + 共有タスクリスト + Mailboxで構成される。5Claude Code Agent Teams 公式ガイド 有効化には環境変数の設定が必要だ:

// settings.json に追加
CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1

// 要件: Claude Code v2.1.32 以上

表示モードは2種類ある:

  • in-process: Shift+Down でチームメイト間を切り替え
  • split-pane: tmux/iTerm2 で各エージェントを並列表示

タスク管理と品質ゲート

タスクは pending → in_progress → completed のフローで管理され、依存関係の自動解決やファイルロック競合防止機能を備える。5Claude Code Agent Teams 公式ガイド

推奨構成は3-5チームメイト、1人あたり5-6タスク。品質ゲートとして TeammateIdle / TaskCompleted Hooksで自動チェックを組み込める。

最適なユースケース

  • 並列コードレビュー: Security / Performance / Tests を別々のエージェントが同時にレビュー
  • 競合仮説デバッグ: 複数のエージェントが異なる仮説を同時に検証
  • クロスレイヤー開発: フロントエンド・バックエンド・インフラを並列で開発
注意: Agent Teamsは実験的機能（experimental）。セッション復旧不可、ネストチーム不可、リーダー固定という制限がある。本番利用には注意が必要。5Claude Code Agent Teams 公式ガイド

Agent TeamsがSubagentsより優れている点として、最も重要なものは？

  • コンテキストコストが低い

階層的な拡張設計

Claude Codeのアーキテクチャは、常時ON（CLAUDE.md）からオンデマンド（Skills）、外部接続（MCP）、並列処理（Subagents/Agent Teams）、自動化（Hooks）、配布（Plugins）へと段階的に拡張できる設計になっている。1Anthropic — Claude Code 公式ドキュメント

コンテキスト管理の最適化

各機能がコンテキストコストを意識して設計されている点は注目に値する。Skillsは低コスト、Hooksはゼロコスト、Subagentsは隔離コンテキストで動作する。トークンコストがチームサイズに比例するAgent Teamsでは、コスト最適化戦略の策定が重要になる。1Anthropic — Claude Code 公式ドキュメント5Claude Code Agent Teams 公式ガイド

Claude Codeの各拡張レイヤーにおいて、コンテキスト管理の設計方針として正しいのは？

  • 全てのレイヤーが同じコンテキストコストで動作する
  • 外部サービス接続（MCP）のコンテキストコストが最も低い

Claude Codeの設計思想として、各レイヤーのコンテキストコストが最適化されています。Hooksは決定論的スクリプトでLLM不要（コスト0）、Subagentsは隔離コンテキストで動作し親のコンテキストを消費しません。

導入ロードマップ

段階的に導入することで、リスクを抑えながらエージェントチームの恩恵を受けられる。

  1. Step 1: 基盤整備 — CLAUDE.md + Skills でプロジェクト知識を整備する。まずはAIが参照できる永続コンテキストの土台を作る。1Anthropic — Claude Code 公式ドキュメント
  2. Step 2: 外部接続 — MCPで業務ツール（DB、Slack、GitHub等）を接続する。AIが外部データにアクセスできる環境を構築。3MCP（Model Context Protocol）仕様
  3. Step 3: 小規模並列 — Subagentsで小規模な並列処理を試行する。単純な分割統治パターンで効果を検証。
  4. Step 4: チーム編成 — Agent Teamsで大規模な探索・レビュータスクに挑戦する。並列コードレビューや競合仮説デバッグを試す。5Claude Code Agent Teams 公式ガイド
  5. Step 5: 組織展開 — Coworkのプラグインで非開発チームへの展開を検討する。法務・営業・マーケティングチームへの導入。6Anthropic — Claude Cowork リサーチプレビュー
